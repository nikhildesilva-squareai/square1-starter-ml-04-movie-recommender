# Data dictionary — `ratings.csv` + `movies.csv`

A movie-ratings log for building a recommender. **Synthetic — generated by Square 1 AI, free for learners.** ~18,000 ratings across 600 users and 300 movies, plus a small movie catalogue for context.

> ⚠️ There is a **real collaborative-filtering signal** in here: users with
> similar taste rate movies alike, so similar movies get similar ratings from
> the same user. A correct CF model (item-kNN or matrix factorisation) recovers
> it and beats a global-mean baseline. Don't try to reverse-engineer the
> generator — let the model find the structure. **Clean first:** a handful of
> duplicate `(user_id, movie_id)` rows exist (keep the **latest** by timestamp)
> and a few ratings fall **outside 1–5** (drop them).

## `ratings.csv` — one row per rating event

| Column | Type | Description |
|---|---|---|
| `user_id` | int | User identifier (1–600). |
| `movie_id` | int | Movie identifier; joins to `movies.csv` (1–300). |
| `rating` | int | The user's rating, **1–5**. A few dirty rows are 0 or 6 — **drop them**. |
| `timestamp` | datetime | When the rating was given (`YYYY-MM-DD HH:MM:SS`). Use it to keep the latest of any duplicate `(user_id, movie_id)`. |

## `movies.csv` — one row per movie (context / cold-start, optional)

| Column | Type | Description |
|---|---|---|
| `movie_id` | int | Movie identifier (1–300). |
| `title` | string | Movie title. |
| `genres` | string | One or two genres, pipe-separated, e.g. `Action\|SciFi`. Context only — not all genres appear in every user's history. |

**What to build:** hold out a portion of the ratings as a test set, fit a collaborative-filtering model on the rest (item-/user-kNN or matrix factorisation), and predict the held-out ratings. Report **RMSE vs a global-mean baseline** and **top-N relevance vs a popularity baseline**, then serve top-N recommendations from a Flask endpoint. The matrix is **sparse** (each user rates a small slice of the catalogue) — that's realistic and part of the challenge.
